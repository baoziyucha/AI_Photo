## 二、P0-2 任务拆解(表情与眨眼 + 真假笑)

### P0-2-T1 · 数据模型扩展
- **文件**:[CameraGuidanceModels.kt](app/src/main/java/com/yuxiang/aiphoto/model/CameraGuidanceModels.kt)
- **内容**:
  1. 新增枚举 `ExpressionState { NATURAL_SMILE, FAKE_SMILE, AWKWARD, NO_SMILE, UNKNOWN }`
  2. `RetryReason` 新增 `BLINKED`(眨眼重拍理由)+ 对应 `toDisplayText()`:「眨眼了,再来一张」
  3. `GuidanceFrame` 已在 P0-1 扩展字段,此处无需再改
- **验收**:编译通过,新枚举可被 GuidanceEngine 引用

### P0-2-T2 · 表情评估器(核心逻辑)
- **文件**:[GuidanceEngine.kt](app/src/main/java/com/yuxiang/aiphoto/analysis/GuidanceEngine.kt)
- **位置**:新增 `private fun evaluateExpression(smiling: Float?, leftEye: Float?, rightEye: Float?): ExpressionState`
- **规则**(`eyeOpen = min(leftEye, rightEye)`):

| smile | eyeOpen | 判断 | 返回 |
|---|---|---|---|
| > 0.7 | ∈ [0.4, 0.8] | 真笑(杜兴笑) | `NATURAL_SMILE` |
| > 0.5 | > 0.9 | 假笑 | `FAKE_SMILE` |
| ∈ [0.2, 0.5] | > 0.9 | 尴尬笑 | `AWKWARD` |
| < 0.2 | 任意 | 没笑 | `NO_SMILE` |

- 任一字段为 null → `UNKNOWN`(降级不评估)
- **验收**:单元测试覆盖 4 种组合 + null 降级

### P0-2-T3 · 眨眼拦截(强制 NOT_READY)
- **文件**:[GuidanceEngine.kt](app/src/main/java/com/yuxiang/aiphoto/analysis/GuidanceEngine.kt)
- **位置**:`calculateCaptureReadiness`([L234-L267](app/src/main/java/com/yuxiang/aiphoto/analysis/GuidanceEngine.kt#L234-L267))开头
- **内容**:`if (min(leftEye, rightEye) < 0.3f) return CaptureReadiness.NOT_READY`(在 `subjectBox == null` 判断之后)
- **验收**:眨眼时 READY 不触发,日志可见「blinked → NOT_READY」

### P0-2-T4 · READY 评分接入表情分
- **文件**:[GuidanceEngine.kt](app/src/main/java/com/yuxiang/aiphoto/analysis/GuidanceEngine.kt)
- **位置**:`calculateCaptureReadiness` + 评分权重常量([L41-L44](app/src/main/java/com/yuxiang/aiphoto/analysis/GuidanceEngine.kt#L41-L44))
- **内容**:
  1. 权重调整:`WEIGHT_COMPOSITION=0.35`、`WEIGHT_BRIGHTNESS=0.25`、`WEIGHT_TILT=0.15`、`WEIGHT_ZOOM=0.10`、`WEIGHT_EXPRESSION=0.15`(从原四项按比例下调,总和=1.0)
  2. 新增 `evaluateExpressionScore(state): Float`:`NATURAL_SMILE→1.0`、`FAKE_SMILE→0.4`、`AWKWARD→0.3`、`NO_SMILE→0.5`、`UNKNOWN→0.7`(降级不惩罚)
  3. `UNKNOWN` 时不纳入评分(按原四项归一化),避免无脸场景误伤
- **验收**:真笑时 READY 阈值更易达成;眨眼已被 T3 拦截;无脸场景评分不异动

### P0-2-T5 · 表情文案分支
- **文件**:[GuidanceEngine.kt](app/src/main/java/com/yuxiang/aiphoto/analysis/GuidanceEngine.kt)
- **位置**:`recommendationFor`([L317-L366](app/src/main/java/com/yuxiang/aiphoto/analysis/GuidanceEngine.kt#L317-L366)),在 `facePitchAdvice` 分支**之后**、`when (sceneType)` **之前**插入
- **内容**:

| 状态 | 文案 |
|---|---|
| 眨眼 | 「你眨眼了,再来一张」 |
| NATURAL_SMILE | 「笑容很自然,拍!」(不阻断,落回场景文案) |
| FAKE_SMILE | 「放松点,想想开心的事,真笑一下」 |
| AWKWARD | 「深呼吸,放松脸部」 |
| NO_SMILE 持续>2s | 「可以笑一下」(需配合时间窗,可用帧计数近似) |

- **协作点**:此处插入新分支,P0-3 改的是下游 `portraitRecommendation` 内部,两者在 `recommendationFor` 的调用链上不重叠,合并时按「P0-2 分支在前、P0-3 改下游」顺序即可
- **验收**:眨眼/假笑/真笑文案正确触发;真笑时不阻断 READY 文案

### P0-2-T6 · 单元测试
- **文件**:[GuidanceEngineTest.kt](app/src/test/java/com/yuxiang/aiphoto/analysis/GuidanceEngineTest.kt)
- **用例**:
  1. 眨眼(eyeOpen<0.3)→ NOT_READY
  2. 真笑(smile>0.7 + eyeOpen∈[0.4,0.8])→ 表情分满分
  3. 假笑(smile>0.5 + eyeOpen>0.9)→ 文案含「真笑」

## 三、P0-3 任务拆解(三分法 + 视线留白 + 负空间)

### P0-3-T1 · 视线方向判断
- **文件**:[GuidanceEngine.kt](app/src/main/java/com/yuxiang/aiphoto/analysis/GuidanceEngine.kt) + [CameraGuidanceModels.kt](app/src/main/java/com/yuxiang/aiphoto/model/CameraGuidanceModels.kt)
- **内容**:新增 `private fun gazeDirection(headEulerY: Float?): GazeDirection`,返回 `LOOK_LEFT`/`LOOK_RIGHT`/`FORWARD`/`UNKNOWN`
  - `|Y| < 5°` → `FORWARD`
  - `Y > 5°` → `LOOK_LEFT`(脸朝左,留白在左)
  - `Y < -5°` → `LOOK_RIGHT`(脸朝右,留白在右)
  - null → `UNKNOWN`(回退居中策略)
- **新增枚举**:`GazeDirection`(放 CameraGuidanceModels.kt,与 P0-2 的 T1 无冲突)
- **验收**:单元测试覆盖三向 + null

### P0-3-T2 · 目标框动态计算
- **文件**:[GuidanceEngine.kt](app/src/main/java/com/yuxiang/aiphoto/analysis/GuidanceEngine.kt)
- **位置**:`computeTargetZone`([L181-L202](app/src/main/java/com/yuxiang/aiphoto/analysis/GuidanceEngine.kt#L181-L202))
- **内容**:
  1. `TARGET_CENTER_X = 0.5f` 改为动态:`LOOK_RIGHT → 0.33f`、`LOOK_LEFT → 0.67f`、`FORWARD/UNKNOWN → 0.5f`
  2. `build()` 需把 `headEulerY` 传入 `computeTargetZone`(新增参数)
- **验收**:脸朝右时目标框移到左侧 1/3;朝左时移到右侧 1/3

### P0-3-T3 · 三分法评分(GuidanceEngine 侧)
- **文件**:[GuidanceEngine.kt](app/src/main/java/com/yuxiang/aiphoto/analysis/GuidanceEngine.kt)
- **位置**:`evaluateComposition`([L269-L287](app/src/main/java/com/yuxiang/aiphoto/analysis/GuidanceEngine.kt#L269-L287))
- **内容**:
  - 废弃 `CENTER_IDEAL_MIN/MAX` 等居中阈值
  - 新逻辑:主体在三分点(0.33±0.08 或 0.67±0.08)+ 视线朝向留白 → 1.0
  - 主体在三分点但视线与留白矛盾(看墙)→ 0.4
  - 主体正中(0.5±0.05)→ 0.8(保留正中为合法构图)
  - 其他 → 0.2
- **验收**:偏左 1/3 看右 → 满分;偏左 1/3 看左(看墙)→ 0.4

### P0-3-T4 · 方向提示调整(不再强制纠正回中心)
- **文件**:[GuidanceEngine.kt](app/src/main/java/com/yuxiang/aiphoto/analysis/GuidanceEngine.kt)
- **位置**:`determineDirectionHint`([L204-L232](app/src/main/java/com/yuxiang/aiphoto/analysis/GuidanceEngine.kt#L204-L232))
- **内容**:主体在三分点 + 视线朝向留白 → 不返回 `MOVE_LEFT/MOVE_RIGHT`(不纠正);仅当主体贴边或视线矛盾时才提示
- **验收**:偏左 1/3 看右 → 无「镜头向右」误导箭头

### P0-3-T5 · 人像文案调整
- **文件**:[GuidanceEngine.kt](app/src/main/java/com/yuxiang/aiphoto/analysis/GuidanceEngine.kt)
- **位置**:`portraitRecommendation`([L415-L432](app/src/main/java/com/yuxiang/aiphoto/analysis/GuidanceEngine.kt#L415-L432))
- **内容**:移除「把人物放回中间」类文案;三分点 + 视线留白正确 → 「构图稳定,可以拍摄」;视线看墙 → 「让脸转向画面深处」
- **协作点**:P0-2 在上游 `recommendationFor` 插入表情分支,P0-3 改本方法内部,合并时无冲突(调用链上下游)
- **验收**:偏左看向右 → 「可以拍摄」;偏左看向左 → 「让脸转向画面深处」

### P0-3-T6 · PhotoScorer 三分法 + 负空间修正
- **文件**:[PhotoScorer.kt](app/src/main/java/com/yuxiang/aiphoto/analysis/PhotoScorer.kt)
- **位置**:`evaluateComposition`([L54-L113](app/src/main/java/com/yuxiang/aiphoto/analysis/PhotoScorer.kt#L54-L113))
- **内容**:
  1. 居中评分([L91-L104](app/src/main/java/com/yuxiang/aiphoto/analysis/PhotoScorer.kt#L91-L104))改为三分法:三分点 + 视线留白 → 满分 + `strengths.add("三分法构图,视线留白到位")`;正中 → 0.8
